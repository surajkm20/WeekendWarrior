2. Ingestion Flow — python ingest.py
1
Load Documents from ./docs/
PyPDFLoader reads each PDF page as a separate document. Also supports .docx and .txt. Each document carries metadata: source filename + page number.
2
Split into Chunks
RecursiveCharacterTextSplitter splits text into chunks of 1000 chars with 150 char overlap. Overlap prevents losing context at boundaries. Tries to split on paragraphs → sentences → words.
3
Generate Embeddings
Each chunk is sent to nomic-embed-text via Ollama. The model returns a vector of 768 numbers representing the semantic meaning of that chunk. Similar meaning = similar vector direction.
4
Store in ChromaDB
Vectors + original text + metadata are persisted to ./vectorstore/ on disk. ChromaDB uses HNSW index for fast similarity search. This step only runs once per document set.
3. Query Flow — what happens when you ask a question
1
User types a question in the browser
Chainlit captures the message and sends it to on_message() in app.py via WebSocket.
2
Contextualize the Question (if chat history exists)
Follow-up questions like "what about the premium?" are rephrased into standalone questions using the LLM + chat history. e.g. → "What is the premium amount in the policy?" This ensures the retriever understands the question without needing history.
3
Vector Similarity Search in ChromaDB
The standalone question is converted to a vector using nomic-embed-text. ChromaDB finds the top 4 most similar chunks by comparing vectors. Returns the raw text of those chunks + their source metadata.
4
Build the Prompt
The answer prompt is assembled with 3 parts: chat_history (last 6 turns) + context (retrieved chunks) + question (original user question). Strict instruction: answer ONLY from the context.
5
LLM Generates Answer (streamed)
llama3.2:3b runs locally via Ollama. The answer is streamed token by token back through LangChain → Chainlit → WebSocket → Browser. User sees text appearing in real time.
6
Show Source Citations + Save History
Source filenames and page numbers are appended below the answer. The question + answer are saved to chat_history (capped at last 6 turns) for use in the next question.

7. Setup & Run Commands

One-time Setup

brew install python@3.12 ollama
ollama pull llama3.2:3b
ollama pull nomic-embed-text
python3.12 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

Every Time You Use It

# Terminal 1 — keep running
ollama serve

# Terminal 2
source .venv/bin/activate
python ingest.py        # only needed after adding new docs
chainlit run app.py     # open http://localhost:8000
